# Spec: 用户头像上传

- ID: F001
- 状态: frozen
- 关联宪法: P1, P2, P6（非功能需求）

## 1. 背景与问题
用户需要上传并更换头像。当前无此能力，影响个人资料完整度。

## 2. 用户故事
- US1: 作为登录用户，我想上传头像图片，以便在资料页展示。

## 3. 验收标准（EARS 风格）
- AC1: 当用户上传 ≤5MB 的 jpg/png 时，系统应当存储并返回可访问 URL。
- AC2: 当文件超限或类型非法时，系统应当返回 400 且不写入存储。
- AC3: 当存储后端不可用时，系统应当返回 503 并记录告警，不得吞错误。

## 4. 非功能需求（80% 问题清单）
- 安全: 校验 MIME 与魔数；防止路径穿越；鉴权后才允许上传。
- 可观测: 上传成功/失败计数与耗时埋点。
- 限流: 单用户每分钟 ≤10 次。
- 审计: 记录 who/when/对象 key。

## 5. 范围与不做
- In scope: 上传、校验、存储、返回 URL。
- Out of scope: 头像裁剪 / CDN 预热。

## 6. 风险
- 大文件占用带宽 → 由限流 + 大小上限缓解。
