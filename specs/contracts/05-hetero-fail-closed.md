# 契约 05 · 异构评审 fail-closed

## 异构的可验证定义

**评审模型的提供商/模型族 ≠ 被审产物作者的模型族,且评审过程可留痕。** 当前实现位点 = shell-out `cursor-agent` 或 `codex`;若某工具底层可路由回作者同族模型(如 cursor 选用 Claude),该次评审**不算异构**——留痕中必须记录足以判断模型族的信息。

## 硬规则

1. **judge 阶段缺真异构 → 退出码 2 转人审。绝不静默退回作者模型自审。** 依据:同质评审的盲区是系统性共享的,作者与评审者同族 = 循环自证。(背景注:本项目实测中作者方修复多次被异构评审抓出自看不到的洞——经验佐证,非规则依据。)更强的作者模型(Fable 5)不豁免。
2. **评审者身份留痕**:记录评审工具、请求参数(如 `-p --mode ask`、是否指定 `--model`)。**工具可观测字段优先;模型自报只作辅助声明**——工具不提供可观测字段时,留痕注明"身份为自报"+ 请求参数,不得编造。
3. 留痕位置:receipt 的 `reviewer` 字段(契约 06)或 `REVIEW-*.md`/PR 描述。
4. 评审纪律:先 steelman 再挑刺;被驳倒就改;辩论结论(接受/部分/反驳+理由)入留痕。

## 钉死方式

- **结构性**:M1 CLI 不调任何模型(契约 04)→ "作者模型自评自过"在 CLI 层无通道可走;
- **流程审计**:每次异构评审产物按第 2 条留痕,缺留痕的"评审"在契约变更/PR 流程中不被承认;
- judge skill 文案内置"异构缺席 → 转人审"分支——**这是实现位点,不是强制机制**(契约 01:会话内构件不算强制),强制兜底在 CI 重跑 gate。
