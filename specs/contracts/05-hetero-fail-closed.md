# 契约 05 · 异构评审 fail-closed

1. **judge 阶段缺真异构(跨厂商模型:cursor-agent / codex)→ 退出码 2 转人审。绝不静默退回 Claude-审-Claude。** Fable 5(或任何更强的 Claude)不豁免——同模型盲区是系统性的(实测:本人 fix 连续 3 次被 cursor 抓出自看不到的洞)。
2. **评审者身份留痕**:每次异构评审的产物必须记录——评审工具、请求参数(如 `-p --mode ask`、是否指定 `--model`)、评审者自报的模型身份;**工具不自报身份时,留痕注明"身份未自报"+ 请求参数**,不得编造。
3. 留痕位置:`specs/<id>/gates/` 内的 receipt(契约 06 的 `actor`/`reviewer` 字段)或 PR 描述。
4. 评审纪律:先 steelman 再挑刺;被驳倒就改;辩论结论(接受/部分/反驳+理由)入留痕。

## 钉死方式

- judge skill(M2)文案内置"异构缺席 → 输出『异构缺席,转人审』"分支,不提供静默回退路径;
- M1 `gate-judge` 本身不调模型(契约 04),三态升级逻辑里没有"Claude 自评自过"通道可走。
