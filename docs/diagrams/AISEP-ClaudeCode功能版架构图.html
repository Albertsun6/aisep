AISEP 用 Claude Code 实现 —— 功能版架构(三视角)

这是"抛开所有 fail-closed 强制约束、只看功能能不能跑通"的架构。核心心智:外层 SDK supervisor 是骨架(定序/并行/resume)、CC 引擎是每阶段的执行力(角色子代理+技能+hooks+宪法)、aiforge 纯 Python 核是大脑(方法论判断,直接复用不重写)、MCP 接外部、OTel 记账。唯一非原生处:真·跨模型异构靠 shell-out cursor/codex。

读法:Driver(外层)是总指挥——它不写代码,只"派活给角色子代理 → 产物交给 aiforge 核判过没过 → 该选优时请异构桥 → 每步记账"。注意 judge 那步会出 CC 进程调 cursor/codex(全图唯一非原生处)。
三视角分别回答:东西怎么摆(图1)、活怎么流(图2)、谁指挥谁(图3)。这是功能蓝图,不含强制脚手架(managed 锁 / HMAC sink / 进程外审批 / attestation —— 那些是"防 AI 掀翻"的,功能版不需要)。
