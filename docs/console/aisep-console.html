把"在聊天里记住六问"换成"结构化表单填六字段"。填完右侧实时生成 intent.md,复制/下载交给会话跑 gate-intent。这一页就是用来解锁 R1 的——你不用预先知道要答什么;不会写就照实写一句现状即可,门禁只检查有没有填、不评判措辞。
feature-id 必填
本次需求的短标识(小写 kebab-case)。例:auction-mvp
track 已选 standard
需求量级 — 决定走全流程还是短路径。
fast
微变更(如"调整页面位置"),R1+R5 短路径
standard
单个特性,走全 R1-R5
epic
史诗级(如"做个 ERP"),R2 后拆子特性
problem 必填
为什么做?给谁解决什么痛点?现在这件事是怎么发生的(线下/Excel/旧系统/全新)?——这句也决定是不是隐性棕地。
users 必填
谁用?各角色 + 大致量级。
成功指标 必填
做完后什么"可观察的现象"出现算成功?要能被看见,不是"提升体验"这种虚的。
non-goals 必填
明确不做什么?(支付清结算?实名认证?直播?App?)——这是范围的种子,越明确越防 AI 臆想。
appetite 必填
愿意投多少?(几人日/几周)——它决定第一版切多大,不是估算而是"意愿上限"。
source 可选
原始材料出处(会议纪要/聊天记录/一句话)。仅留痕,不影响门禁。
