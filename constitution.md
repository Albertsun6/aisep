# AIForge Constitution（项目宪法）

> 规格驱动开发的第 0 步。这是 AI agent 与人类协作产出企业软件时**不可逾越**的工程原则。下游每个 spec / plan / tasks 都必须对照本宪法验收。原则只可追加（编号递增），不可删改核心语义。

## P1 — 规格先于代码
任何功能实现前必须存在被冻结的 `spec.md`。代码服务于规格，而非规格迁就代码。无 spec 的实现视为不合规，门禁拒绝。

## P2 — 可追溯
spec → plan → tasks → 代码 → 测试必须形成可追溯链路。每个任务都能回答"它满足哪条需求 / 哪条验收标准"。

## P3 — 质量门禁不可绕过
所有产出必须经四层门禁：IDE/pre-commit → PR（覆盖率 + SAST + Agent-as-Judge）→ CI/CD（权威、不可绕过）→ 生产 canary + 自动回滚。CI 失败的 diff 任何人不得合并。

## P4 — 最小权限与安全停机
agent 默认最小权限，按任务申请。面对无法安全完成的 state-mutating 任务（改环境/删数据/改配置），必须安全停机（safe-halt）并上报，禁止"改环境 / 吞错误 / 谎报完成"。

## P5 — Blast Radius 受限
单次自主变更的文件数 / 影响面有硬上限，超限自动降级为需人审。失败时影响面可度量、可回滚。

## P6 — 非功能需求是一等公民
功能性 80% 不等于完成。错误处理、安全、可观测、合规、限流、审计（即"另外 20%"）必须由显式门禁强制，agent 不会自觉补齐。

## P7 — 认知债治理
产出速度超过人类 review 速度时，强制三文件 review（精读 diff 最大的 3 个文件）+ 按风险分级 review。测试通过 ≠ 被理解；高风险变更必须有人能解释"为什么这样改"。

## P8 — 全量审计
每个决策、工具调用、权限授予都写入审计 trail，可复现、可审查。审计追踪一律追加写、写入后不可篡改（可追加补充说明）；决策"改对过的错"如实留痕，不得事后洗白。

## P9 — 选型可替换
模型 / 编排框架 / runtime 通过适配层接入，保持可替换，避免被单一供应商锁定。

## P10 — 来源诚实
关键结论（安全 / 正确性 / 架构）必须可追溯到具体来源：代码、文档、可执行验证、或被命名的评审者；不得产生"普遍认为 / 共识是"式无源断言。低置信度推断必须显式标注，不与已验证事实混同。强制钩子：契约 01（诚实标注建议层 vs 强制层）+ 契约 05（异构评审）。

## P11 — 异构独立验证
关键结论不可由作者模型自审自过；必须经独立视角（跨模型 lens）验证，各视角坚守立场、不互相同化，收敛过快须触发反方复审（防虚假共识与共享盲区）。强制钩子：契约 05（异构 fail-closed，CI 重跑）。

> P10/P11 收编自 2026-06 老库 AISEP MetaP §15-17 的元认知原则。P12（记忆不可变）经评估其语义已被 P8（追加写不可篡改）+ 契约 06（receipt 链冻结）+ 契约 08（audit append-only）完全覆盖，故并入 P8 不单列（避免无新增强制的空原则）。
